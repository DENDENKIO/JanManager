# JanManager 最新技術仕様書

**最終更新日**: 2026-04-06 (全体刷新：スキャニングUI一本化・自動仮登録・ステータス復元対応版)

---

## 1. システム概要
JanManagerは、商品のJANコードをスキャン・管理し、AI（Gemini/Perplexity）を活用した商品情報の自動補完、包装単位の親子関係管理（単品・パック・ケース）、発注支援およびグループ管理を行うAndroidアプリケーションです。

### 1.1 設計方針
- **常時フォーカスUI**: HIDスキャナー使用時の待ち時間をゼロにするため、常にJAN入力欄へフォーカス。
- **データ欠落防止**: マスタ未登録の商品でも即座に「仮登録」し、後からAIで情報を補完。
- **入力の厳密化**: すべてのJAN入力において半角数字への強制正規化を実施。

---

## 2. 技術スタック
- **Language**: Kotlin 1.9+ / Jetpack Compose (Material 3)
- **Database**: Room (SQLCipherによる暗号化等、セキュアなデータ管理に対応可)
- **DI**: Hilt / Dagger
- **Networking**: WebView (AI連携用 JavaScript Injection)
- **Scanning**: CameraX (モバイルカメラ) + Bluetooth HID (外部スキャナー)

## 3. スキャン・入力仕様

### 3.1 統一されたスキャンUI (Scan UI Unification)
「スキャナーモード（隠し）」と「手入力モード」を廃止し、**可視状態の単一入力フィールド**へ一本化。
- **常時フォーカス制御**: 500ms ごとに `focusRequester.requestFocus()` を送出。
- **Enterキーハンドリング**: スキャナーが送信する末尾の `Enter` キーを `onKeyEvent` でキャッチし、入力確定と後続処理（登録・検索）を即座に実行。
- **IME干渉防止**: `KeyboardType.Number` を使用し、且つ `KeyEvent` を優先捕捉することで全角変換による入力途切れを防止。

### 3.2 入力正規化 (`Normalizer`)
すべての JAN 入力（スキャン、検索、紐づけ、編集）において `toHalfWidth()` を適用。
- 全角数字を半角数字へ変換。
- 全角空白の削除、半角空白のトリミング。
- 14桁のITFコードは自動的に13桁のJANコード（単品）へ読み替え（ビジネスロジックによる）。

## 4. AI自動情報取得（詳細仕様）

### 4.1 自動貼り付け・送信ロジック
`AiWebViewInteractor.kt` を通じて、Web上のAIサービスへJANを自動的に bridge。

1.  **プロンプト注入**: JSの `document.execCommand('insertText', ...)` を使用。これにより、React等の仮想DOMが入力イベントを確実に検出し、送信ボタンが有効化される。
2.  **自動クリック**: 送信アイコン（SVGやButton）を CSS セレクタで特定し、`click()`。
3.  **フォールバック送信**: ボタンが特定できない場合、入力要素に対して `Enter` の `KeyboardEvent` を発行。

### 4.2 応答待機と安定判定 (Stabilization)
AIの回答（逐次表示）が完了したことを、テキスト変化の監視で判定。
- **変化監視**: 500ms ごとに取得。
- **完了判定**: **6回連続（計3秒）** で文字列の長さが変わらない場合、生成完了とみなす。
- **JSON抽出**: 回答テキスト内の「最後のJSONブロック」を抽出し、`ProductMaster` へマッピング。

## 5. データ管理とライフサイクル

### 5.1 未登録商品の自動仮登録 (Auto-Stub Creation)
発注支援スキャンおよびグループスキャンにて、マスタにない JAN が読み込まれた際の挙動。
1.  **商品マスタ検索**: JAN で直接検索。
2.  **包装単位検索**: 包装単位（パック/ケース）テーブルを検索し、単品 JAN を取得して再検索。
3.  **自動仮登録**: どちらでも見つからない場合、`info_fetched = false` として商品マスタに新規挿入。
4.  **メーカー名補完**: 過去に登録された JAN プリフィックスからメーカー名を自動引用（メーカーキャッシュ連携）。

### 5.2 ステータス復元と紐づけ解除
商品ライフサイクルにおける誤操作防止。
- **終売（DISCONTINUED）取り消し**: 商品詳細画面からステータスを `ACTIVE` へ復元。
- **リニューアル紐づけ解除**: 紐づいた新旧 JAN の関係性を双方向でクリアし、両商品を `ACTIVE` 状態へ戻す。

### 5.3 包装単位の簡素化
パック・ケースの管理をバーコードと種別のみに限定。
- **入数の削除**: UI から入数を撤廃。管理上の複雑さを排除し、スキャンによる同一商品特定のみを目的とする。

## 6. 主要なディレクトリ構成
- `ui/scan/`: 通常のスキャン・履歴。
- `ui/order/`: 発注支援スキャン（仮登録ロジック含む）。
- `ui/group/`: グループ管理（常時フォーカス対応版）。
- `ui/product/`: 商品詳細・編集・ステータス復元。
- `data/repository/`: 商品・グループ・メーカーキャッシュのデータアクセス。
- `util/Normalizer.kt`: 半角数字正規化ユーティリティ。

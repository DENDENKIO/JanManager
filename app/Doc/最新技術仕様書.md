# JanManager 最新技術仕様書

**最終更新日**: 2026-04-06

---

## 1. システム概要
JanManagerは、商品のJANコード（バーコード）をスキャン・管理し、AI（Gemini/Perplexity）を活用して商品情報を自動的に補完・登録するAndroidアプリケーションです。

### 1.1 設計思想
- **ローカルファースト**: 商品データはRoomデータベースに保存され、オフラインでも閲覧・管理が可能。
- **AIアシスト**: 未登録商品の情報をWeb上のAIサービスから自動取得し、手動入力の手間を最小限に抑える。
- **階層型管理**: 単品（Piece）、パック（Pack）、ケース（Case）という包装単位の親子関係をサポート。

---

## 2. 技術スタック
- **Language**: Kotlin 1.9+
- **UI**: Jetpack Compose (Material 3)
- **Architecture**: MVVM + Clean Architecture 準拠
- **DI**: Hilt / Dagger
- **Database**: Room Persistence Library
- **External Integration**: WebView (JavaScript Injection方式)
- **Media**: CameraX (バーコードスキャン)

## 3. 機能仕様

### 3.1 スキャンモード
`ScanViewModel` にて `ScanModeTab` によって制御。

- **連続モード (CONTINUOUS)**: 連続してスキャンし、履歴をリスト表示。登録の有無を確認する用途。
- **確認モード (CONFIRM)**: スキャンした商品の詳細を表示。未登録または情報不足の場合、ここからAI取得を開始する。
- **包装紐づけモード (LINKAGE)**: 
    - 単品、パック、ケースのJANコードを順番にスキャンし、それらの親子関係をデータベースに登録する。
    - 入数（数量）の設定も同時に行う。

### 3.2 商品管理 (Product Master)
- **ステータス管理**: 「販売中」「終売」「更新済」の3状態。
- **リニューアル管理**: 旧JANコードと新JANコードの紐づけ。
- **メーカー管理**: JANコードの上位桁からメーカー名を自動抽出、またはAIから取得。

### 3.3 グループ管理
- 商品を任意のグループ（棚、カテゴリ等）に分類。
- 各グループにはイメージカラーを設定可能。

## 4. AI自動情報取得メカニズム（詳細仕様）

本システムの核心となる、AIを用いた商品情報自動インポートの仕組みです。

### 4.1 通信インターフェース (`JanBridge`)
WebView内で動くJavaScriptから、Android側のKotlinコードを呼び出すためのブリッジです。
- **`onResult(text: String?)`**: AIの応答テキストを取得した際に呼ばれる。
- **`onError(message: String?)`**: エラー発生時に呼ばれる。

### 4.2 プロンプト生成 (`AiPromptBuilder`)
スキャンしたJANコードを元に、AIに対して以下のような「厳密な出力フォーマット」を指示するプロンプトを生成します。
- 出力は必ず指定された JSON 形式（`{ "jan_code": "...", "product_name": "...", ... }`）にすること。
- 商品が見つからない場合は `{ "not_found": true }` と出力すること。

### 4.3 自動貼り付け（インジェクション）ロジック
`AiWebViewInteractor.kt` 内の JavaScript によって実行されます。

1.  **入力フィールドの特定**: GeminiやPerplexityの入力欄（textarea等）を、設定されたCSSセレクタで検索。
2.  **プロンプト注入**: 
    - 一般的な `value` 代入ではなく、`document.execCommand('insertText', ...)` を優先使用。これにより、Webサイト側の入力イベント（onChange等）を確実に発火させる。
    - プロンプト文字列はBase64エンコードしてJSに渡し、デコードすることでエスケープ文字によるJSエラーを防止。
3.  **自動送信**:
    - 送信ボタンをセレクタで特定し `click()`。
    - ボタンが見つからない、または無効な場合は `KeyboardEvent('Enter')` を入力欄に発行するフォールバックを実装。

### 4.4 応答待機と安定判定 (Stabilization)
AIの回答が「動的に生成（逐次表示）」されるため、完了を判定するロジックを持っています。

1.  **新規応答の検知**: JS側で、送信前のレスポンス要素数と送信後の要素数を比較し、新しい要素が出現するまで待機。
2.  **テキスト安定判定**:
    - 500msごとに応答テキストの長さをチェック。
    - **6回連続（計3秒）** で長さが変わらなければ「生成完了」とみなす。
    - **`not_found` 早期終了**: テキスト内に `not_found` キーワードが含まれた場合、安定を待たずに即座に終了。
    - **強制終了保護**: 10秒間変化がない、または90秒経過した場合はタイムアウトとして処理。

### 4.5 パースとバリデーション (`AiResponseParser`)
AIから返された自然言語混じりのテキストから、有効なデータを抽出します。

1.  **JSON抽出**: テキスト内の `{` と `}` の対応をカウントし、**最後のJSONブロック**を抽出（以前の会話履歴が含まれる場合への対策）。
2.  **不一致チェック**: AIが回答した `jan_code` が、スキャンしたJANコードと一致するか検証。
3.  **正規化**: 
    - `Normalizer` を通じて、全角半角、不要な空白を整形。
    - メーカー名や商品名が英字のみの場合等に備え、カナ変換（`toKana`）を適用。

## 5. データモデル (Database Schema)

### 5.1 `product_master`
| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | Long (PK) | 内部ID |
| `jan_code` | String (Unique) | 商品のJANコード |
| `maker_name` | String | メーカー名 |
| `product_name` | String | 商品名 |
| `spec` | String | 規格（容量・重量等） |
| `status` | Enum | ACTIVE, DISCONTINUED, RENEWED |
| `info_fetched` | Boolean | AIから取得済みか |
| `info_source`| Enum | NONE, AI_GEMINI, AI_PERPLEXITY |

### 5.2 `package_unit`
| カラム名 | 型 | 説明 |
| :--- | :--- | :--- |
| `id` | Long (PK) | 内部ID |
| `product_id` | Long (FK) | `product_master.id` への参照 |
| `barcode` | String | 包装単位のJAN/ITFコード |
| `package_type`| Enum | PIECE, PACK, CASE |
| `quantity` | Int | 入数 |

---

## 6. 特筆すべき実装上のTips
- **ITFコード対応**: 14桁のITFコードをスキャンした場合、自動的に13桁のJANコードに変換して管理。
- **メーカーキャッシュ**: 同じJANプリフィックスの商品を一度登録すると、以降はメーカー名が自動補完される仕組み。
- **WebView制御**: AIサービスのURL遷移や、ログイン状況の維持など、ユーザーの利便性を損なわない制御を `AiWebViewWrapper` で行っている。

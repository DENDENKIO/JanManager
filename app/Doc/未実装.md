## カテゴリ1: 発注スキャン（OrderScan）

### ✅ 1-1. EAN-8 / ITF-14 バーコード未対応（実装完了）
**現状:** `BarcodeGenerator.kt` に `generateBarcodeBitmap()` を追加し、EAN-8, EAN-13, ITF の自動判別に対応。  
**問題:** EAN-8（8桁）スキャン時にバーコード生成が失敗する。ITF-14は設定で有効化できるがバーコード生成未対応。  
**必要な実装:**
```kotlin
// BarcodeGenerator.kt に追加
fun generateEan8Bitmap(code: String, width: Int, height: Int): ImageBitmap?
fun generateItf14Bitmap(code: String, width: Int, height: Int): ImageBitmap?
```
- `OrderListScreen.kt` の `generateEan13Bitmap()` 呼び出し箇所をコード長で分岐
- `JanCodeUtil.detectCodeType()` の結果を `OrderListViewModel` に渡して適切なジェネレータを選択

---

### ✅ 1-2. 重複JAN排除ロジックが警告のみで阻止しない（実装完了）
**現状:** `OrderScanViewModel.processBarcode()` で重複を検知した場合、DBへの追加をスキップし、UIに重複警告を通知するように修正。
**問題:** 同じJANが複数回スキャンされると一覧に重複表示される不具合を解消。

---

### ✅ 1-3. 終売品警告ダイアログ未実装（実装完了）
**現状:** `OrderScanViewModel` に `pendingDiscontinuedProduct` ステートを追加。終売品スキャン時に `OrderScanScreen.kt` で `AlertDialog` を表示し、ユーザーが承認した場合のみ追加されるフローを実装。
**現状:** `OrderScanViewModel` で `isDiscontinued = true` はセットするが、**UIに警告Dialogが存在しない**。  
**問題:** 終売品をスキャンしても気づかずに発注リストに入る。  
**必要な実装:**
```kotlin
// OrderScanScreen.kt に追加
if (uiState.isDiscontinued) {
    AlertDialog(
        title = { Text("⚠️ 終売品") },
        text = { Text("この商品は終売です。発注リストに追加しますか？") },
        confirmButton = { Button(onClick = { viewModel.confirmDiscontinued() }) { Text("追加") } },
        dismissButton = { Button(onClick = { viewModel.cancelDiscontinued() }) { Text("キャンセル") } }
    )
}
```
- `OrderScanViewModel` に `confirmDiscontinued()` / `cancelDiscontinued()` メソッドを追加
- 確認後のみ `addItemToSession()` を実行

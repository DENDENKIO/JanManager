# 発注スキャンシステム — Androidアプリ実装依頼プロンプト

---

## 📋 プロジェクト概要

スーパー・小売店の売り場で使用する **Androidアプリ** の一機能として、**発注スキャンシステム** を実装してほしい。  
売り場スタッフがBluetoothスキャナーでJANコードを読み取り、そのリストを発注機で読み取れる形で画面表示する一連の仕組みである。

- **言語**: Kotlin
- **UI**: Jetpack Compose
- **DB**: Room ORM
- **対象OS**: Android 10以上

---

## 🗄️ データベース設計

### テーブル1: ScanSession（スキャンセッション）

売り場ごと・日付ごとのスキャン作業単位を管理する。

```kotlin
@Entity(tableName = "scan_sessions")
data class ScanSession(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val sessionName: String,          // 例: "4/5 菓子棚"
    val createdAt: Long,              // System.currentTimeMillis()
    val status: SessionStatus         // OPEN / COMPLETED
)

enum class SessionStatus { OPEN, COMPLETED }
```

### テーブル2: ScanItem（スキャンアイテム）

各セッションで読み取ったJANコードの記録。**数量は持たない**。重複排除はアプリ側で行う。

```kotlin
@Entity(
    tableName = "scan_items",
    foreignKeys = [ForeignKey(
        entity = ScanSession::class,
        parentColumns = ["id"],
        childColumns = ["sessionId"],
        onDelete = ForeignKey.CASCADE
    )]
)
data class ScanItem(
    @PrimaryKey(autoGenerate = true) val id: Long = 0,
    val sessionId: Long,              // ScanSession.id への外部キー
    val productId: Long?,             // ProductMaster.id (未ヒット時はnull)
    val scannedBarcode: String,       // スキャンしたJANコード（生値）
    val scanOrder: Int,               // スキャン順番（表示順用）
    val scannedAt: Long               // System.currentTimeMillis()
)
```

### テーブル3: ProductMaster（商品マスタ）— 参照のみ

発注スキャンシステムはこのテーブルを参照するだけで、書き込みは行わない。

```kotlin
@Entity(tableName = "product_master")
data class ProductMaster(
    @PrimaryKey val id: Long,
    val janCode: String,
    val productName: String,
    val makerName: String,
    val spec: String,                 // 例: "350ml缶"
    val isDiscontinued: Boolean       // 終売フラグ
    // packageUnit等は別テーブルでも可
)
```

## 📱 画面1: 発注スキャン画面（ScanScreen）

### 目的
売り場でBluetoothスキャナーを使い、連続してJANコードを読み取り、セッションとして保存する。

### UI構成

```
┌─────────────────────────────────┐
│  [←] 発注スキャン                │  ← TopAppBar
│  セッション: "4/5 菓子棚"         │
├─────────────────────────────────┤
│  [不可視テキストフィールド]          │  ← Bluetoothスキャナー入力受付（常にフォーカス）
├─────────────────────────────────┤
│  スキャン済み商品リスト             │
│  ┌──────────────────────────┐  │
│  │ ① 商品名                  │  │
│  │   メーカー名 / 規格        │  │
│  │   JAN: 4901234567890     │  │
│  └──────────────────────────┘  │
│  ...（LazyColumn）              │
├─────────────────────────────────┤
│  [完了→一覧へ]                   │  ← 下部ボタン
└─────────────────────────────────┘
```

### 機能仕様

#### スキャン入力処理

```
Bluetoothスキャナー → HID入力（Enterキー付き） → TextField → onValueChange → processBarcode()
```

1. `TextField` を画面に配置するが **visually hidden** にする（`alpha = 0f` または `Modifier.size(1.dp)`）
2. 画面表示時に `LaunchedEffect` で即フォーカスを当てる（`FocusRequester`を使用）
3. `\n`（Enter）を検知したらバーコード文字列を `processBarcode()` へ渡す

#### processBarcode() の処理フロー

```
入力バーコード
  │
  ├─ ITF設定がOFF かつ 14桁 → スルー（何もしない）
  │
  ├─ 既にセッション内に同一JANあり
  │     → Toast: "既にスキャン済みです"
  │
  ├─ ProductMasterに存在しない
  │     → Toast: "商品マスタに未登録のバーコードです"
  │
  ├─ 終売品（isDiscontinued == true）
  │     → AlertDialog: "この商品は終売です。追加しますか？"
  │         [追加する] → ScanItemに追加
  │         [キャンセル] → 何もしない
  │
  └─ 通常ヒット
        → ScanItemに追加 → リストを更新 → 成功フィードバック（振動等）
```

#### 完了ボタン

- タップ → ScanSession の `status` を `COMPLETED` に更新
- 発注一覧画面（OrderListScreen）へ遷移（`sessionId` を引数で渡す）

## 📱 画面2: 発注一覧画面（OrderListScreen）

### 目的
スキャン済みのJANコードを発注機で読み取れるよう、バーコード画像付きの一覧をスマホ画面に表示する。  
発注機のバーコードリーダーで画面を読み取ることで発注操作を行う。

### 2つの表示モード

画面上部のタブまたはトグルで切り替える。

---

#### モードA: 一覧モード（ListMode）

**複数商品を縦スクロールで表示する。**

```
┌─────────────────────────────────┐
│  [←] 発注一覧   [一覧|フォーカス] │
├─────────────────────────────────┤
│  ┌──────────────────────────┐  │
│  │ ▌▌▌▌▌▌▌ ← バーコード画像  │  │  ← タップで非表示/再表示（トグル）
│  │ 4901234567890            │  │
│  │ キリンビール株式会社        │  │
│  │ キリン一番搾り 350ml缶     │  │
│  └──────────────────────────┘  │
│  ┌──────────────────────────┐  │
│  │ ✓ 読み取り済み            │  │  ← タップ後の表示（背景グレー）
│  │ 4902102001015            │  │
│  │ アサヒビール株式会社        │  │
│  │ アサヒスーパードライ 350ml缶│  │
│  └──────────────────────────┘  │
│  ...（LazyColumn）              │
└─────────────────────────────────┘
```

**各行の仕様**:
- バーコード画像: ZXing + Compose ラッパーで EAN-13 生成
- バーコード部分を **タップ** すると:
  - バーコード画像 → 非表示
  - 「✓ 読み取り済み」テキストをグレーで表示
  - 行背景を薄いグレーに変更
  - 再タップで元のバーコード表示に戻す（**トグル動作**）
- **数量は一切表示しない**（"数量: 1" などは出さない）

---

#### モードB: フォーカスモード（FocusMode）

**1画面1商品で大きなバーコードを表示する。**

```
┌─────────────────────────────────┐
│  [←] フォーカス   3 / 12        │  ← インデックス表示
│  読み取り済み: 2 / 12            │
├─────────────────────────────────┤
│                                 │
│  キリンビール株式会社              │
│  キリン一番搾り 350ml缶           │
│                                 │
│  ┌───────────────────────────┐ │
│  │                           │ │
│  │  ▌▌▌▌▌▌▌▌▌▌▌▌▌▌▌        │ │  ← 大きなバーコード（タップで非表示）
│  │                           │ │
│  │  4901234567890            │ │
│  │                           │ │
│  └───────────────────────────┘ │
│                                 │
│  ← スワイプ →（前後の商品へ移動）  │
└─────────────────────────────────┘
```

**仕様**:
- バーコードを **タップ** すると:
  - バーコードが非表示になり、中央に「✓ 読み取り済み」を大きく表示
  - 再タップでバーコード再表示（トグル）
- **左右スワイプ** で前後の商品へ移動
- 全商品読み取り完了時に完了ダイアログ表示
- 画面表示時に **スマホ輝度を自動最大化**（発注機の読み取り精度向上のため）

## 🔧 バーコード生成（ZXing）

```kotlin
// build.gradle.kts (app)
implementation("com.google.zxing:core:3.5.3")

// バーコード生成ユーティリティ
fun generateBarcodeBitmap(
    barcodeValue: String,
    width: Int = 600,
    height: Int = 200,
    format: BarcodeFormat = BarcodeFormat.EAN_13
): Bitmap {
    val hints = mapOf(EncodeHintType.MARGIN to 1)
    val bitMatrix = MultiFormatWriter().encode(barcodeValue, format, width, height, hints)
    val bmp = Bitmap.createBitmap(width, height, Bitmap.Config.RGB_565)
    for (x in 0 until width) {
        for (y in 0 until height) {
            bmp.setPixel(x, y, if (bitMatrix[x, y]) Color.BLACK else Color.WHITE)
        }
    }
    return bmp
}
```

## 💡 画面輝度の自動最大化（フォーカスモード用）

```kotlin
// OrderListScreen.kt (Composable内)
val context = LocalContext.current

DisposableEffect(isFocusMode) {
    if (isFocusMode) {
        val window = (context as? Activity)?.window
        val originalBrightness = window?.attributes?.screenBrightness ?: -1f
        window?.attributes = window?.attributes?.also {
            it.screenBrightness = 1.0f  // 最大輝度
        }
        onDispose {
            window?.attributes = window?.attributes?.also {
                it.screenBrightness = originalBrightness
            }
        }
    } else {
        onDispose {}
    }
}
```

## ✅ 実装時の注意点（重要）

| 項目 | 仕様 |
|------|------|
| 数量カラム | **一切作らない**。ScanItemに数量フィールドは不要 |
| 重複スキャン | 同一セッション内で同一JANを2回スキャンしたらToastのみ、DBには追加しない |
| ITF（14桁）| 設定でITF無効の場合、14桁コードは完全スルー（エラーも出さない） |
| 終売品 | スキャン時に警告ダイアログ。ユーザーが「追加する」を選んだ場合のみ登録 |
| バーコード読み取り済みトグル | DBに保存しない。画面内のStateで管理（セッション中の一時状態） |
| 輝度変更 | フォーカスモード離脱時に必ず元の輝度に戻す（DisposableEffect使用） |
| フォーカス維持 | スキャン画面では常にHidden TextFieldにフォーカスを当て続ける |


```
    │
    ├─ [新規セッション] → セッション名ダイアログ → ScanScreen(sessionId)
    │
    └─ [既存セッション選択] → OrderListScreen(sessionId)

    │
    └─ [完了→一覧へ] → OrderListScreen(sessionId)

    ├─ [一覧モード] ←→ [フォーカスモード]（タブ切り替え）
```

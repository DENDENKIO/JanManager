# 自動貼り付け・取り込みの仕組み

> **対象バージョン**: 変更点6（2026-04-06）時点での正しい設計

---

## 概要

JanManager の AI 自動取得機能は、以下の 4 段階で構成されています。

```
[1] ターゲット要素の特定
        ↓
[2] プロンプトの注入（Injection）
        ↓
[3] 送信ボタンのクリック
        ↓
[4] レスポンスのポーリング取得 → JSON 解析 → DB 保存
```

---

## 段階 1：ターゲット要素の特定（フォールバックチェーン）

AI サイトの UI は頻繁に変更されるため、複数のセレクタを優先順に試行するフォールバックチェーンを採用しています。

### 入力欄セレクタ

#### Gemini（Quill エディタ）

```javascript
var selectors = [
  'div.ql-editor[contenteditable="true"]',  // 最優先（Quill 専用クラス）
  'div[contenteditable="true"][role="textbox"]',
  'rich-textarea div[contenteditable]',
  'div[contenteditable="true"]'
];
```

#### Perplexity（Lexical エディタ）

```javascript
var selectors = [
  '#ask-input',                               // 最優先（固有 ID）
  '[data-lexical-editor="true"]',             // Lexical 属性
  'div[contenteditable="true"][role="textbox"]',
  'textarea[placeholder]',
  'div[contenteditable="true"]'
];
```

### 選択ロジック

```javascript
var el = null;
for (var i = 0; i < selectors.length; i++) {
  var found = document.querySelector(selectors[i]);
  if (found) { el = found; break; }
}
if (!el) return false;  // 全セレクタ失敗 → false を返す
```

> **ブラウザ互換性**: `for...of` ではなく `for (var i = 0; ...)` を使用。旧式 WebView でのエラーを防いでいます。

## 段階 2：プロンプトの注入（変更点6 における正しい実装）

単純な `innerText` 直接代入は React / Lexical の仮想 DOM を無視するため無効です。  
以下の序でエディタに「正しく入力された」と認識させます。

```javascript
// ステップ 1: フォーカスと既存テキストのクリア
el.focus();
document.execCommand('selectAll', false, null);
document.execCommand('delete', false, null);

// ステップ 2: DataTransfer + ClipboardEvent('paste') で注入
// → Lexical / Quill が最も確実に反応する方法
var dt = new DataTransfer();
dt.setData('text/plain', prompt);
var pasteEvent = new ClipboardEvent('paste', {
  bubbles: true,
  cancelable: true,
  clipboardData: dt
});
el.dispatchEvent(pasteEvent);

// ステップ 3: innerText が空の場合のフォールバック（直接代入）
if (!el.innerText || !el.innerText.trim()) {
  el.innerText = prompt;
}

// ステップ 4: エディタ内部状態を更新するイベント発火
el.dispatchEvent(new InputEvent('beforeinput', {
  inputType: 'insertText',
  data: prompt,
  bubbles: true
}));
el.dispatchEvent(new Event('input',  { bubbles: true }));
el.dispatchEvent(new Event('change', { bubbles: true }));

return (el.innerText && el.innerText.trim().length > 0);  // 成功判定
```

### 旧実装との比較

| 項目 | 変更点5 以前 | 変更点6（現在） |
|---|---|---|
| 主要注入方法 | `execCommand('insertText')` | `DataTransfer` + `ClipboardEvent('paste')` |
| innerText 直接代入 | 無条件に実行 | paste 後に innerText が空の場合のみ |
| ループ構文 | `for...of` | `for (var i = 0; ...)` |
| Lexical 互換性 | 不安定 | 安定（paste イベント経由） |

#### Perplexity

```javascript
  'button[data-testid="send-button"]',  // 追加
  'button[type="submit"]'
];
// ※ SVG 子要素でのクリックは廃止した
```

### クリックイベント（3 段階発火）

フレームワークのクリックハンドラを確実に起動するため、`mousedown` → `mouseup` → `click` の順に発火します。

```javascript
var btn = null;
for (var i = 0; i < btns.length; i++) {
  var b = document.querySelector(btns[i]);
  if (b) { btn = b; break; }
}

if (btn) {
  var types = ['mousedown', 'mouseup', 'click'];
  for (var j = 0; j < types.length; j++) {
    btn.dispatchEvent(new MouseEvent(types[j], { bubbles: true }));
  }
  return true;
} else {
  // フォールバック: Enter キーイベント
  if (el) {
    el.dispatchEvent(new KeyboardEvent('keydown', {
      key: 'Enter', code: 'Enter', keyCode: 13, bubbles: true
    }));
  }
  return true;
}
```

## 段階 4：レスポンスのポーリング取得

### ポーリング処理（AiFetchViewModel.kt）

```kotlin
// 変更点6 時点: 最大30回（30秒）待機
repeat(30) {
    delay(1000L)
    val result = evaluateJsSync(WebViewJsHelper.getExtractResponseJs())
    if (!result.isNullOrBlank() && result != "null") {
        val cleaned = result
            .removePrefix("\"")
            .removeSuffix("\"")
            .replace("\\n", "\n")
            .replace("\\t", "\t")      // 追加
            .replace("\\\\", "\\")    // 追加
        if (cleaned.length > 5) {   // 5 文字以下は生成途中の断片とみなし無視
            // → JSON 解析処理へ
            return@repeat
        }
    }
}
```

## evaluateJsSync の設計（変更点6 における正しい実装）

```kotlin
private suspend fun evaluateJsSync(script: String): String? {
    val wv = webView ?: return null          // null なら即座に return（ハング防止）
    return try {
        withTimeout(10_000L) {               // 10 秒タイムアウト
            suspendCancellableCoroutine { continuation ->
                wv.post {
                    wv.evaluateJavascript(script) { result ->
                        if (continuation.isActive) {
                            continuation.resumeWith(Result.success(result))
                        }
                    }
                }
            }
        }
    } catch (e: TimeoutCancellationException) {
        null                                 // タイムアウト → null を返す
    }
}
```

### JS 返り値の判定（重要）

`evaluateJavascript` は JavaScript の `true` を **JSON 文字列 `"true"` （引用符付き）** として返します。

```kotlin
// 変更点5 以前（バグ）: == "true" のみ → 常に失敗
val success = injectResult == "true"

// 変更点6（修正）: 引用符なし・ありの両パターンに対応
val success = injectResult == "true" || injectResult == "\"true\""
```

## リトライ設計

注入・送信ともに最大 **3 回** のリトライを実装しています。

```kotlin
var injectSuccess = false
for (attempt in 1..3) {
    val result = evaluateJsSync(WebViewJsHelper.getInjectPromptJs(prompt))
    injectSuccess = result == "true" || result == "\"true\""
    if (injectSuccess) break
    delay(1000L * attempt)  // 指数バックオフ（1秒・2秒・3秒）
}
if (!injectSuccess) {
    // ステータスバーにエラー表示
    return
}
```

## 設定画面 WebView のスクロール設計（変更点6 で追加）

設定画面（`SettingsScreen.kt`）の AI サイトログイン WebView は、Compose の `verticalScroll` と競合しスクロールできない問題がありました。

### 解決策

```kotlin
// AndroidView 内の WebView 設定
webView.isVerticalScrollBarEnabled = true
webView.isHorizontalScrollBarEnabled = true
webView.settings.setSupportZoom(true)
webView.settings.builtInZoomControls = true
webView.settings.displayZoomControls = false  // ズーム UI は非表示（ピンチズームのみ）

// Compose レイアウト
Box(
    modifier = Modifier
        .fillMaxWidth()
        .weight(1f)  // 固定 height(450.dp) ではなく weight で伸縮
) {
    AndroidView(factory = { webView }, modifier = Modifier.fillMaxSize())
}
// ※ nestedScroll は付与しない → WebView 内スクロールをネイティブに委譲
```

## 他機能への水平展開の指針

新しい Web サイトや自動化機能を実装する場合も、以下の設計パターンを守ることで不安定要素を排除できます。

1. **セレクタの集約**: `WebViewJsHelper` にフォールバックリストを集約
2. **判定ロジック**: `evaluateJsSync` の戻り値は引用符付き/なしの両方を常にチェック
3. **リトライ**: 全ステップに最大3回のリトライを実装
4. **ステータス表示**: ユーザーに現在の試行回数を提示し安心感を与える
